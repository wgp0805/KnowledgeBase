## 定义

JWT 无状态认证是指服务端不存储任何 Token 信息，通过验证 Token 自身的签名来确认其有效性。这是 JWT 最核心的设计理念。

## 核心原理

### 验证流程

1. 客户端携带 Token 请求
2. 服务端用密钥验证 Signature
3. 验证通过后从 Payload 提取用户信息
4. 全程不查数据库、不查 Redis

### 优势

- **可水平扩展**：服务端无状态，任意节点都能处理请求
- **跨服务身份传递**：网关验签后，下游服务直接从 JWT 解析 userId
- **减少存储依赖**：不需要 Session 存储

### 局限性

- **无法主动吊销**：已签发的 Token 在过期前一直有效
- **无法即时失效**：用户密码被盗、账号被封禁时，Token 仍可使用
- **无法控制会话**：不能限制同时在线设备数

## 适用场景

| 场景 | 是否适合纯 JWT |
|------|---------------|
| 微服务间身份传递 | ✅ 适合 |
| 一次性凭证（邮箱验证、密码重置） | ✅ 适合 |
| 内部小工具（安全要求低） | ✅ 适合 |
| 需要踢人下线的业务系统 | ❌ 不适合 |
| 需要封禁用户的系统 | ❌ 不适合 |
