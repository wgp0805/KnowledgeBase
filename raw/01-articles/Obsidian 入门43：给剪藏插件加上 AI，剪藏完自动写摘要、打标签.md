# Obsidian 入门43：给剪藏插件加上 AI，剪藏完自动写摘要、打标签

# AI 摘要
**核心要点：**
1. 深度学习在医学影像分析中显著提高了诊断准确率和效率，尤其在CT、MRI等图像识别任务中表现优异。
2. 文章介绍了卷积神经网络（CNN）和U-Net等模型在病灶分割、分类和检测中的具体应用案例。
3. 数据标注困难、模型可解释性不足以及医疗数据隐私问题仍是当前主要挑战。
4. 未来趋势包括结合多模态数据、联邦学习以及自监督学习，以降低对标注数据的依赖。

# 原文内容
简单说：你可以在剪藏的时候，让 AI 自动读一遍网页内容，然后按你的要求生成一些东西，直接写进笔记里。

### 第一步：启用 AI 解释器

点浏览器工具栏的 Web Clipper 图标，再点右上角的齿轮进设置。

找到「解释器」区域，打开「启用解释器」开关。

![](https://pics1.baidu.com/feed/8b82b9014a90f603fb82c5385363140ab051ed26.jpeg@f_auto?token=ca62572e02eef2eb78071130376ff87c)

### 第二步：添加 AI 模型

Web Clipper 内置了好几个 AI 服务商的预设：Anthropic（Claude）、OpenAI（GPT）、Google Gemini、DeepSeek，甚至支持 Ollama 本地模型。

![](https://pics2.baidu.com/feed/472309f7905298222886df25bdbbdcda0b46d426.jpeg@f_auto?token=2c406235add2c807d6d74dbfb58977b4)

操作步骤：

1. 点「添加服务商」，选择你要用的服务商
2. 粘贴你的 API Key（DeepSeek或者千问）
3. 点「添加模型」，选刚才添加的服务商，填一个模型 ID（比如 DeepSeek 填 `deepseek-chat`，Gemini 填 `gemini-2.0-flash`）
4. 给模型起个显示名，方便以后识别

**推荐用小型模型**，速度快，剪藏不需要太强的推理能力。

以我配置的硅基流动的 API为例，因为在提供商的列表中没有硅基流动，所以我选择了 OpenAI, 因为现在绝大多数的提供商都支持 OpenAI 兼容格式。

接着在基础URL这里，填写你选择的 API 平台提供的基础URL。一般来说在 API key对应的页面有相关的文档的，可以在那里查看。

![](https://pics0.baidu.com/feed/0dd7912397dda1445fb43128d9c677b30df4866a.jpeg@f_auto?token=56d67228c030ae883a7e7cc220ff38a8)

最后面把你的 API key 填入，然后保存。

在模型的位置，点击添加按钮，提供商选择你刚刚添加的那个。因为我选择的是OpenAI，所以他默认提供了一些模型供你选择。

以我用的GLM-4.5-Air为例，选择自定义，将模型 ID 填好。模型名称你可以自己自定义给他命名，自己能区分就好。

![](https://pics1.baidu.com/feed/8601a18b87d6277f73611fb94249b821e824fc47.jpeg@f_auto?token=8044ed61bc854ab637d4efe4273b3fa6)

那么，怎么去拿到正确的模型 ID 呢？以我使用的硅基流动平台为例，在模型广场，点击你想要配置的模型。

右侧会出现这个模型的详细信息，点击右上角的复制按钮，把这个模型 ID 粘贴到 Web Clipper插件的输入框中即可

![](https://pics6.baidu.com/feed/ac4bd11373f0820221b3c6ec208a5cfcaa641b46.jpeg@f_auto?token=89e3e0eb10238bacda087c5b3850db13)

如果也想使用硅基流动平台的模型和 API 的朋友，可以使用下面的链接去注册：  
https://cloud.siliconflow.cn/i/9R5h9Cgh

硅基流动上有各个提供商的模型（例如DeepSeek，Kimi，千问等），可选择的余地比较大。一个 API key就可以使用多加模型。

### 第三步：在模板里写 AI 指令

回到模板设置，在「笔记内容」标签页里，用 `请用简洁的语言总结文章的核心要点，突出最新进展和主要挑战。` 这种双引号语法写 AI 指令。一定要用 2 个英文的大括号将提示词括起来，AI 才能识别这个是你给的指令。

点一下「解释」，AI 就会读一遍网页，按你的提示词生成内容。等几秒钟，预览区域就会显示结果。满意了再点「添加到 Obsidian」保存。

### 效果是什么样的？

第一次看到效果的时候，我愣了一下。一篇三千字的英文技术文章，剪藏完打开笔记：

- `description` 属性自动填了一句话摘要
- `tags` 属性自动选好了 2-3 个标签
- 笔记正文最前面整整齐齐列了五条中文要点

我扫了一眼就知道这篇文章值不值得细读。

以前收藏文章是「先存再说」，现在是「先看摘要再决定要不要读」。

### 长文摘要

如果文章很长，可以让 AI 生成更详细的摘要：

1

**主旨：** 深度学习技术正在推动医学影像诊断走向自动化与精准化，但仍需克服数据、隐私和可解释性难题。
**关键点：**
1. **性能突破**：CNN和U-Net在肺部结节检测、视网膜病变识别等任务中达到甚至超过人类专家水平。
2. **应用场景**：涵盖肿瘤分割、骨折检测、器官定位等多个临床方向，显著缩短诊断时间。
3. **数据瓶颈**：医疗影像标注成本高、专家稀缺，制约模型泛化能力。
4. **可解释性需求**：临床医生对黑箱模型不信任，需发展注意力机制等方法提升透明度。
5. **隐私保护**：联邦学习允许跨医院协作训练而不共享原始数据，成为重要解决方案。
6. **未来方向**：自监督学习利用未标注数据预训练，降低对标注需求；多模态融合增强诊断鲁棒性。
7. **伦理考量**：算法偏见需谨慎处理，确保不同人群得到公平诊断。

### 常见问题

**会不会变慢？**  
会多等几秒钟，但说实话，大多数文章你根本感觉不到差别。如上面建议的，尽量配置flash 或者 Air 模型，这样的话速度会快很多。如果配置思考能力较强的模型，速度相对应的就会较慢。

**能同时用好几个模型吗？**  
能。添加多个模型后，剪藏时下拉框选一个就行。

**AI 生成的不好怎么办？**  
重新点「解释」就行，AI 会重新生成。也可以调调提示词，让它换种方式总结。

**API Key 安全吗？**  
Key 只存在浏览器本地，不会上传到任何服务器。不过建议别把 Key 分享给别人。

1

**主旨：** 深度学习技术正在推动医学影像诊断走向自动化与精准化，但仍需克服数据、隐私和可解释性难题。
**关键点：**
1. **性能突破**：CNN和U-Net在肺部结节检测、视网膜病变识别等任务中达到甚至超过人类专家水平。
2. **应用场景**：涵盖肿瘤分割、骨折检测、器官定位等多个临床方向，显著缩短诊断时间。
3. **数据瓶颈**：医疗影像标注成本高、专家稀缺，制约模型泛化能力。
4. **可解释性需求**：临床医生对黑箱模型不信任，需发展注意力机制等方法提升透明度。
5. **隐私保护**：联邦学习允许跨医院协作训练而不共享原始数据，成为重要解决方案。
6. **未来方向**：自监督学习利用未标注数据预训练，降低对标注需求；多模态融合增强诊断鲁棒性。
7. **伦理考量**：算法偏见需谨慎处理，确保不同人群得到公平诊断。
